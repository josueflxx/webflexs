FLEXS API v1

La API usa el encabezado Authorization: Token <token>. En cuentas con varias empresas agrega ?company_id=ID en cada solicitud.

Los tokens heredan empresas, roles y permisos del usuario. Desactivar la cuenta revoca su acceso.

Recursos

MetodoRutaUso
POST/api/v1/auth/token/Obtiene token con usuario y contraseña.
GET/api/v1/catalog/products/Productos; admite q, category, supplier e in_stock.
GET/api/v1/clients/Clientes de la empresa activa.
GET/api/v1/orders/Pedidos, estado y totales.
GET/POST/api/v1/webhooks/Lista o crea destinos de webhook.
PATCH/DELETE/api/v1/webhooks/{id}/Edita, rota el secreto o elimina.
GET/api/v1/schema/Esquema legible por herramientas.

Firma de webhooks

Se envian X-FLEXS-Event, X-FLEXS-Event-ID, X-FLEXS-Timestamp y X-FLEXS-Signature. Verifica la firma calculando HMAC-SHA256 sobre <timestamp>.<raw_body> con el secreto del endpoint.

signature = HMAC_SHA256(secret, timestamp + "." + raw_request_body)
expected_header = "sha256=" + signature

Eventos disponibles: order.created, order.status_changed, payment.recorded y fiscal.updated. Las entregas fallidas se reintentan con espera progresiva.
